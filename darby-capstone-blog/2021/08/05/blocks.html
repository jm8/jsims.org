Rendering Code Blocks

Published on Aug 5, 2021

    On the AP Computer Science principles test, there are two different representations for pseudocode, which can be found on the

    The first is Text, which looks like this:

    DISPLAY(expression)

    or

    IF(condition) {
    <block of statements>
}

    The other is Block, which looks like this:

    For my application I have decided that people will write and edit their code in text format, since that is a lot easier than creating a full-on block editor (as seen in Scratch for example).

    However I want people to be able to see the block format, and so I have made it so that it can render the AST into these blocks.

    I have created a AstElement React component, which takes an AST and turns it into an HTML version of the block. For example, to render an if statement, it creates a filled in block, and also contains an indented block. It then recursivley creates more AstElements for each of the things inside it. My code looks something like this:

    const renderBlock = (block: Ast[]) => (<div className="indentedBlock">{block.map(render)}</div>);
const alwaysInParens = (subast: Ast) => <AstElement ast={subast} parens="always" />
// ...
if (ast.type === "if") {
    return <div className="filled">
        <div className="condition">IF {alwaysInParens(ast.condition)}</div>
        {renderBlock(ast.iftrue)}
    </div >
}

    Then I use CSS to style it and make it look like the blocks. For example, for the filled if statement, I give it a border and background and rounded corners:

    .CodeBlock .filled {
    border: 0.10em solid black;
    border-radius: .56em;
    padding: 0.3333em;
    background-color: #c7c8ca;
}
