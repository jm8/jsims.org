Basic UI

Published on Aug 4, 2021

    I want to keep my UI (User Interface) code and my code that runs the language as seperate as possible. However I want a way to be able to try out what I am working on in the interpreter, and what I decided to do was create a basic User Interface that would let me write and parse code.

    Of course I haven’t actually written the parser yet. So write now it’ll just look like this:

    export type Ast = any;
export function parse(code: string): Ast {
    return {
        example: "this is an example AST",
        yourCode: code
    }
}

    But I wrote some React code that creates a UI that will let you edit code, parse it, and show you the AST (Abstract Syntax Tree), which is the result of the parser. This is what it looks like:

    Obviously, it is not pretty (I will make it pretty later), but it will allow me to run my parser and see its output.

    I use a different component Editor or Running to show depending on if we are editing the code or running it.

    if (data.mode == "editing") {
    content = <Editor code={data.code} setCode={(code) => setData({ ...data, code })} onRun={onRun} />
} else if (data.mode == "running") {
    content = <Running code={data.code} ast={data.ast} onClose={() => setData({ mode: "editing", code: data.code })} />
}

    I need to provide these functions for what to do when something changes, as well as the data that they show, like the code, AST, or error message.

    The editor just shows a text field where we can write code and a to run it.

    return (<div>
    <div><button onClick={() => onRun()}>Run</button></div>
    <textarea value={code} onChange={(e) => setCode(e.target.value)} />
</div>

    And when it’s running it will show the code and the AST (in JSON format).

    return (<div>
    <button onClick={e => onClose()}>Close</button>
    <pre><code>{code}</code></pre>
    <pre><code>{JSON.stringify(ast, undefined, 2)}</code></pre>
</div>)

    I will have to change these later, for example to show error messages, and to add buttons to control the execution. But right now I have the overall structure of the app.
