Typescript and React

Published on Aug 3, 2021

    Since my program is going to be on the web, it must be in Javascript. However I am using Typescript. Typescript code gets turned into Javascript code, which is the programming language every web application uses.

    The reason I am using Typescript instead of Javascript is that it in plain Javascript it is really easy to make mistakes like this, and it won’t tell you. It can be really hard to find these kinds of errors.

    2+"hello" // => "2hello"

let x = {a: 2}
x.b // => undefined

    However in Typescript the compiler will tell you as soon as you type it. This will make it and faster to write my code.

    Also, I am choosing to use React, which is a UI framework. It basically makes it a lot easier to write your UI by letting you making components, which lets you easily split up parts of the UI and reuse components. They will automatically react when something changes in their properties. For example, I could make a component called Editor, and do something like this:

    <Editor value={code} />

    And the Editor would re render whenever code changed, and I wouldn’t have to keep track of that myself. This will also help me make code faster and more organized.

    What’s nice about using Typescript and React is I can just do npx create-react-app --template typescript, and it automatically makes all the files I need. Then I can just run npm start and it’ll automatically recompile when I make changes.
